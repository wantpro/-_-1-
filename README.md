# 크롤링 및 수요 분석 도구 모음

공개 웹 커뮤니티, 앱 스토어, 네이버 카페에서 데이터를 수집하고, 반복적으로 나타나는 고민과 주제를 분석하기 위한 도구 모음입니다. 수집 경로별 특성에 맞춰 코드를 분리하고, 수집 결과가 콘텐츠 기획과 수요 분석으로 이어지도록 구성했습니다.

## 구성

| 폴더 | 역할 | 주요 기술 |
| --- | --- | --- |
| `크롤링_차서연/` | 네이버 카페 게시글 목록·본문 수집 및 단어 빈도 분석 | Python, Selenium, pandas, Kiwi |
| `크롤링_최비성/` | 공개 커뮤니티·앱 스토어 수집과 웨딩 주제 수요 분석 | Node.js, Playwright, 스토어 수집 라이브러리 |

## 네이버 카페 수집과 단어 분석

`크롤링_차서연/`은 네이버 카페에서 게시글 목록을 수집한 뒤, 각 게시글의 본문을 보완하고 단어 빈도를 분석하는 순서로 사용합니다.

1. `naver_cafe_list.py` — 게시판 주소를 기준으로 게시글 제목과 링크를 수집합니다.
2. `naver_cafe_body.py` — 수집한 링크를 정리·중복 제거한 뒤 로그인 세션을 활용해 본문을 수집합니다.
3. `word_frequency.py` — 본문에서 명사와 외국어 단어를 추출하고, 불용어를 제외해 게시글별·전체 단어 빈도를 만듭니다.

수집 과정에서 URL 형식을 정규화하고, 잘못된 제목·중복 게시글·접근 제한 게시글을 걸러냅니다. 결과 파일은 기본적으로 사용자 다운로드 폴더에 CSV 형식으로 저장됩니다. 네이버 카페의 이용 약관과 접근 권한을 준수해야 하며, 로그인이나 접근 제한을 우회하지 않습니다.

## 공개 웹·앱 스토어 수집과 웨딩 수요 분석

`크롤링_최비성/`은 공개 웹페이지와 앱 스토어의 원본 자료를 수집하고, 웨딩 관련 게시글에서 반복되는 주제와 콘텐츠 수요를 정리합니다.

- `src/web/` — 블라인드, 디시인사이드, 당근, 스레드의 공개 페이지 수집기입니다. 페이지 렌더링이 필요한 경우 Playwright를 사용합니다.
- `src/stores/` — 구글 플레이와 애플 앱 스토어의 리뷰를 수집합니다. 설치 수가 1만~100만인 중소형 앱과 1~3점 리뷰를 우선 대상으로 삼습니다.
- `src/analysis/` — 수집 문서에서 화면 요소·광고·개인정보를 정리하고, 규칙 기반 토픽 분류와 수요 점수 계산, 콘텐츠 기획안·워드클라우드 생성을 수행합니다.
- `src/shared/` — 수집 속도 조절과 안전한 파일 저장처럼 여러 수집기가 공통으로 쓰는 기능입니다.

웨딩 분석 파이프라인은 생성형 AI나 유료 API에 의존하지 않습니다. 사람이 정한 키워드 사전으로 주제를 분류하고, 게시글 수·질문 비율·고민 표현·반응을 조합해 수요 점수를 계산합니다. 분석 결과는 보고서, 중간 산출물, 실행 기록으로 나눠 저장해 재현과 검토가 가능하도록 합니다.

## 실행 예시

공개 웹·앱 스토어 수집과 분석은 `크롤링_최비성/` 폴더에서 실행합니다.

```bash
cd 크롤링_최비성
npm install

# 공개 웹 수집
npm run web:blind -- --query="결혼,웨딩" --shards=2 --max=30 --interval=2500
npm run web:dcinside -- --query="웨딩 견적" --pages=2

# 앱 스토어 리뷰 수집
npm run stores:google-play
npm run stores:app-store

# 웨딩 주제 수요 분석
npm run analysis:run
npm run analysis:wordcloud
```

네이버 카페 수집은 각 Python 파일 상단의 게시판 주소, 수집 건수, 저장 경로를 목적에 맞게 설정한 뒤 실행합니다. Selenium, pandas, kiwipiepy와 Chrome WebDriver가 필요합니다.

## 데이터와 준수 사항

- 수집 대상은 공개적으로 접근 가능한 페이지와 허용된 API 범위로 한정합니다.
- 로그인·가입·권한이 필요한 콘텐츠는 서비스의 정책과 접근 권한을 따릅니다.
- 수집 속도를 제한하고, 원본 데이터와 분석 결과는 용도별로 분리해 보관합니다.
- 개인정보와 화면 요소, 광고 문구는 분석 전에 가능한 한 제거합니다.
- 수집 결과는 시장·콘텐츠 수요를 이해하기 위한 참고 자료이며, 사실 검증이나 개인 식별을 위한 도구가 아닙니다.
