# Task 05 — 인스타 콘텐츠 백로그 설계

## 목표

상위 수요 토픽을 게시물 아이디어가 아니라, 타깃·문제·형식·근거가 있는 실행 가능한 콘텐츠 백로그로 바꾼다.

## 변환 규칙

토픽 하나당 아래 네 가지를 정리한다.

1. **누구의 어느 순간인가**: 예비부부, 식장 계약 전, 드레스 투어 전처럼 구체화한다.
2. **무슨 질문/불안인가**: 대표 글의 질문을 비식별 요약으로 한 문장으로 쓴다.
3. **무엇을 주는가**: 비교표, 체크리스트, 예산 계산법, 질문 템플릿, 공감형 경험담 중 하나로 정한다.
4. **왜 지금 만들까**: 문서 수·질문 비율·반응 등 Task 04 근거를 연결한다.

## 형식 매핑

| 수요 유형 | 우선 형식 | 예시 산출물 |
|---|---|---|
| 비교·추천 요청 | 캐러셀 | 식장 투어 전 질문 10개 |
| 비용·계약 불안 | 캐러셀/저장형 리일스 | 계약서에서 추가금 확인할 곳 |
| 갈등·의사결정 | 공감형 리일스/캐러셀 | 부모님 의견이 갈릴 때 정할 3가지 |
| 후기·경험 공유 | 리일스/스토리 | 계약 전에 알았으면 좋았을 것 |
| 짧은 사실 질문 | Q&A 카드 | 예산 항목 한 장 설명 |

## 출력

`data/analysis/wedding/report/content-backlog.csv`

```text
priority,topic_id,target_stage,audience,pain_or_question,content_promise,
format,hook,outline,cta,evidence_post_ids,demand_score,status
```

상위 10개 토픽에 대해 토픽당 최소 3개, 총 30개 이상을 만든다. 같은 사실을 제목만 바꾼 중복 콘텐츠는 하나로 합친다.

## 완료 기준

- [ ] 모든 아이디어가 특정 토픽과 수요 근거를 가짐
- [ ] 상위 10개 토픽마다 저장형 1개 이상, 공감/도달형 1개 이상 포함
- [ ] 의료·법률·재정 조언은 개인 사례 일반화나 단정 표현을 피함
- [ ] 원문·작성자·식별 정보를 콘텐츠에 노출하지 않음

## 다음 태스크 입력

우선순위가 매겨진 콘텐츠 백로그와 근거 링크/ID.
