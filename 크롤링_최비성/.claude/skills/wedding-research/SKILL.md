# 웨딩 인스타 콘텐츠 리서치 하네스

## 목적

블라인드 `결혼생활`과 Threads `웨딩` 공개 대화에서 반복되는 질문·고민·후기를 찾아, 인스타그램에서 만들 콘텐츠 주제를 우선순위화한다. 이 하네스는 **AI 없이도 실행 가능한 자동 분석**을 기준으로 한다.

## 작업 순서

| 순서 | 태스크 | 입력 | 핵심 산출물 |
|---:|---|---|---|
| 01 | [크롤링](../wedding-crawl/SKILL.md) | 공개 게시글/API 결과 | 소스별 원본 JSON |
| 02 | [정제·통합](../wedding-clean/SKILL.md) | 원본 JSON | 분석용 CSV·제외 로그 |
| 03 | [자동 토픽 분석](../wedding-topics/SKILL.md) | 분석용 CSV | 글별 토픽·토픽 요약 |
| 04 | [수요 신호 산출](../wedding-demand/SKILL.md) | 토픽 결과·반응 | 우선순위 표 |
| 05 | [인스타 콘텐츠 설계](../wedding-content-plan/SKILL.md) | 우선순위 표·대표 글 | 콘텐츠 백로그 |
| 06 | [검수·리포트](../wedding-report/SKILL.md) | 모든 산출물 | 최종 리포트·재실행 기록 |

## 공통 원칙

- 수집은 공개 글 또는 플랫폼 공식 API로만 한다. 로그인·인증·차단은 우회하지 않는다.
- “글 수”는 수요의 대리 지표일 뿐이다. 질문 비율, 반응, 최근성, 플랫폼 편향을 함께 본다.
- 플랫폼별 수치(조회·댓글·좋아요)는 절대값으로 서로 비교하지 않는다.
- 원문 공개·재배포 대신, 분석 결과에는 최소 인용과 비식별화된 요약만 쓴다.
- 결과물의 기준일과 수집 기간을 항상 함께 기록한다.

## 권장 폴더 구조

```text
data/web/{blind,threads}/            # 원본: 웹 수집기 출력
data/analysis/wedding/report/        # 사람이 열어보는 결과물
data/analysis/wedding/intermediate/  # 단계 간 중간물
data/analysis/wedding/meta/          # 재현·감사 기록
.claude/skills/                      # 파이프라인 태스크 문서
```
