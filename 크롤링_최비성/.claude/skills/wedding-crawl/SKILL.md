# Task 01 — 웨딩 대화 데이터 수집

## 목표

인스타그램 콘텐츠 수요 탐색을 위한 원천 텍스트를 수집한다. 이번 대상은 공개 범위의 블라인드 `결혼생활` 관련 글이다. Threads 전체 공개 검색은 App Review가 필요해 이번 범위에서 제외한다.

이 단계에서는 주제 분류·워드클라우드·AI 분석을 하지 않는다. 수집 원문과 최소 메타데이터를 재현 가능하게 저장하는 것이 완료 조건이다.

## 원칙

- 수집 속도를 제한하고, 오류·차단 시 재시도 폭주를 하지 않는다.
- 게시글 URL·수집 시각·검색어를 남겨 출처와 중복을 추적한다.
- 개인 식별 정보는 분석에 필요하지 않으므로 후속 단계에서 제거한다.

## 수집 범위

| 소스 | 범위 | 검색어 | 목표량 | 저장 위치 |
|---|---|---|---:|---|
| Blind | 공개 `결혼생활` 채널의 게시글. 인증/로그인 게이트 글 제외 | 채널 전체 | 공개 목록 끝까지 | `data/web/blind/` |

검색어별 결과를 따로 저장한다. 같은 글이 여러 검색어로 나올 수 있으므로, 다음 정제 단계에서 URL/ID 기준으로 통합 중복 제거한다.

## 실행 절차

### 1. Blind 공개글 수집

기존 `src/web/blind.js`를 사용한다. 이 수집기는 공개 `결혼생활` 채널을 끝까지 스크롤해 URL을 찾고, 공개 접근 가능한 페이지만 느린 간격으로 렌더링한다.

```bash
node src/web/blind.js \
  --topic-url="https://www.teamblind.com/kr/topics/%EA%B2%B0%ED%98%BC%EC%83%9D%ED%99%9C" \
  --interval=2500
```

- 목록 탐색은 1.2초, 개별 글 렌더링은 2.5초 간격으로 한다.
- 브라우저 의존성이 없으면 `npx playwright install chromium`을 먼저 실행한다.
- `결혼생활` 채널 여부는 저장된 `board` 필드로 확인한다. 다른 공개 채널에 섞인 결과는 삭제하지 말고 `board` 값을 보존한 뒤 후속 정제에서 제외/비교한다.
- 게이트된 글, 본문 추출 실패 글, 빈 텍스트 글 수를 실행 로그에 기록한다.

### 2. Threads 수집 제외

Threads의 전체 공개 게시물 키워드 검색에는 Meta 앱의 `threads_keyword_search` 권한 승인(App Review)이 필요하다. 승인 전에는 인증한 계정이 소유한 게시물만 검색되므로, 이 태스크의 공개 대화 수집 목적에 맞지 않는다.

- Threads는 이번 수집 범위에서 제외한다.
- 브라우저 스크래핑, 세션 쿠키 사용, 비공식 엔드포인트 우회는 하지 않는다.
- `src/web/threads.js`는 삭제하지 않고, 향후 승인된 공식 권한이 생길 때만 사용한다.
- Task 02는 Blind 수집 결과만 입력으로 진행한다.

## 산출물 검수

각 JSON은 `CommunityDoc` 형식이며 최소한 다음 필드를 확인한다.

```text
id, board, title, text, url, author, date
```

수집 후 아래를 점검한다.

1. 파일별 문서 수, 빈 `text` 수, 중복 ID 수를 집계한다.
2. 임의로 각 소스에서 10개씩 열어 제목/본문이 실제 게시글과 맞는지 확인한다.
3. Blind 결과에서 `board`가 결혼생활 관련인 비율을 기록한다.
4. HTML 메뉴, 추천글, 광고 문구처럼 본문이 아닌 텍스트가 과도하게 포함됐는지 확인한다.
5. 각 파일에 검색어와 수집 일시가 남아 있는지 확인한다.

## 완료 기준

- [ ] Blind 검색어별 JSON이 `data/web/blind/`에 저장됨
- [ ] 각 파일의 문서 수·빈 본문 수·게이트 제외 수를 기록함
- [x] Threads는 전체 공개 검색에 App Review가 필요해 이번 범위에서 제외함
- [ ] 표본 검수에서 본문 추출 오류/보일러플레이트 문제를 확인함
- [ ] 중복 제거와 개인정보 최소화가 필요한 항목을 다음 태스크로 넘김
