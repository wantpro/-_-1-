# Task 06 — 결과 검수·리포트·재실행 기록

## 목표

누가 보더라도 “무슨 데이터를, 어떤 기준으로, 어디까지 해석했는지” 알 수 있는 최종 리포트와 재실행 기록을 만든다.

## 리포트 구성

1. **요약**: 표본 수, 기간, 소스, 가장 강한 콘텐츠 기회 3~5개
2. **데이터 한계**: 공개 글 표본 편향, Blind 익명 고민 편향, Threads API 범위·권한, 날짜 결측
3. **토픽 결과**: 상위 토픽, 상위 구문, 대표 질문의 비식별 요약
4. **수요 우선순위**: 산식, 플랫폼별 결과, 해석
5. **콘텐츠 제안**: 우선 제작 10개와 각 근거
6. **다음 수집 가설**: 부족한 주제, 추가할 검색어, 다음 수집 시점

## 시각화 원칙

- 전체 워드클라우드 하나로 결론을 내리지 않는다.
- 토픽별 2~3어절 워드클라우드 또는 상위 구문 막대그래프를 사용한다.
- 수요점수는 구성요소(대화량·질문·고민·반응)를 함께 보여준다.
- 플랫폼별 차트는 분리한다.

## 재현성 기록

- 수집 실행일, 검색어, 파일명, 문서 수
- 정제 제외 건수와 규칙 버전
- 토픽 수, 불용어 파일, 라이브러리 버전, 난수 시드
- 점수 산식과 결측 처리

위 내용은 `data/analysis/wedding/meta/run-manifest.json`에 저장한다. 최종 문서는 `data/analysis/wedding/report/research-report.md`에 작성한다.

## 완료 기준

- [ ] 결론마다 표본·토픽·점수 근거가 연결됨
- [ ] 워드클라우드는 탐색용으로만 사용되고 정량 표가 함께 있음
- [ ] 개인정보·원문 장문 인용이 없음
- [ ] 다른 사람이 원본과 설정으로 같은 결과를 재생성할 수 있음
- [ ] 다음 수집/분석 주기가 명시됨
