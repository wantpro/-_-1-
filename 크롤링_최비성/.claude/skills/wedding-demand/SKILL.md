# Task 04 — 콘텐츠 수요 신호와 우선순위 산출

## 목표

토픽별로 “대화량”이 아니라 인스타 콘텐츠로 만들 만한 문제·질문 수요를 비교한다. 점수는 결정 도구이며 절대적인 시장 규모가 아니다.

## 산출 지표

각 플랫폼 안에서 계산한 뒤 비교한다.

| 지표 | 정의 | 해석 |
|---|---|---|
| 대화량 | 토픽 문서 수 / 전체 문서 수 | 반복적으로 언급되는가 |
| 질문 비율 | `?`, 부탁·추천·어떻게·괜찮아 등 질문 신호가 있는 글 비율 | 해결 콘텐츠 수요가 있는가 |
| 고민 강도 | 불안·갈등·후회·비용 초과·파혼 등 사전 기반 표현 비율 | 절박한 문제인가 |
| 반응 | 댓글 수 또는 조회 수의 플랫폼 내 백분위 | 대화를 유발하는가 |
| 최근성 | 날짜가 있는 글의 최근 30일 비중/증가율 | 지금 다뤄야 하는가 |

## 점수

메타데이터가 충분한 경우 아래 0~100 점수를 사용한다.

```text
수요점수 = 25×대화량_정규화
         + 30×질문비율
         + 25×고민강도
         + 20×반응_플랫폼내정규화
```

- 날짜가 부족하면 최근성은 점수에 넣지 말고 참고 지표로만 표시한다.
- 조회 수가 비정상적으로 파싱된 글은 반응 계산에서 제외한다.
- Threads와 Blind 점수는 합산하지 않고, 소스별 점수와 공통 토픽 여부를 나란히 제시한다.

## 출력

- `data/analysis/wedding/report/topic-demand.csv`
- `data/analysis/wedding/meta/demand-notes.md`

## 완료 기준

- [ ] 토픽별 산식과 누락값 처리 방식이 기록됨
- [ ] 상위 토픽 10개에 대표 질문/고민 구문이 함께 있음
- [ ] 단순 후기·잡담이 상위에 온 경우, 질문/고민 지표와 분리해 해석함
- [ ] 플랫폼 편향과 표본 한계를 리포트에 명시함

## 다음 태스크 입력

상위 토픽, 대표 질문 구문, 수요점수와 소스별 차이.
