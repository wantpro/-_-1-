# Task 03 — AI 없는 자동 토픽 분류

## 목표

사전 라벨링 없이 정제된 게시글을 텍스트 패턴으로 자동 군집화하고, 글마다 `topic_id`를 부여한다. 생성형 AI, 유료 API, 수작업 라벨링은 사용하지 않는다.

## 방법

1. `kiwipiepy`로 명사·동사 어간·복합 명사를 추출한다.
2. 한 글 전체와 함께 2~3어절 n-gram을 만든다. `식장 투어`, `계약 취소`, `예산 초과` 같은 구문을 보존한다.
3. 불용어(결혼, 웨딩, 너무, 그냥 등)와 플랫폼 UI 문구를 제외한다. 브랜드명은 별도 열에 보존하고 기본 토픽 입력에서는 제외한다.
4. TF-IDF 행렬에 NMF를 적용한다. 시작 토픽 수는 문서 수에 따라 8~15개로 하고, 같은 데이터에서 8·10·12·15개를 각각 실행한다.
5. 각 토픽에 상위 구문 15개, 대표 글 5개, 문서 수를 출력한다. 사람 친화적 주제명은 강제하지 않고 `topic_00`처럼 ID를 유지한다.

## 선택 기준

- 한 토픽의 상위 구문이 지나치게 일반적이거나, 대표 글이 서로 무관하면 토픽 수를 줄이거나 불용어를 보완한다.
- 거의 모든 글을 차지하는 토픽은 토픽 수를 늘리거나 공통어를 더 제거한다.
- 3문서 미만 토픽은 `other`로 합치되 원래 결과는 보존한다.
- 짧은 Threads 글은 정보량이 적으므로 토픽 신뢰도를 별도 기록한다.

## 출력

- `data/analysis/wedding/intermediate/posts-topics.csv` — `post_id, topic_id, topic_weight` 추가
- `data/analysis/wedding/intermediate/topic-summary.csv` — 토픽별 문서 수, 상위 구문, 대표 글 ID
- `data/analysis/wedding/meta/topic-model-config.json` — 토픽 수, 불용어, 실행일, 라이브러리 버전

## 완료 기준

- [ ] 전체 분석 글에 `topic_id`가 있음
- [ ] 토픽마다 상위 구문과 대표 글이 생성됨
- [ ] 최소 두 가지 토픽 수 결과를 비교하고 선택 근거를 기록함
- [ ] 원문 라벨이나 LLM/API 호출 없이 재실행 가능함
